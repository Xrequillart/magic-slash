Comment ça marcheHow it works

Magic Slash repose sur deux mécanismes clés de Claude Code : les Slash Commands et les serveurs MCP.Magic Slash relies on two key Claude Code mechanisms: Slash Commands and MCP servers.

Slash Commands

Les slash commands sont des fichiers Markdown stockés dans ~/.claude/commands/. Quand tu tapes /start, Claude Code lit le fichier start.md et exécute les instructions qu'il contient.Slash commands are Markdown files stored in ~/.claude/commands/. When you type /start, Claude Code reads the start.md file and executes the instructions it contains.

Chaque commande Magic Slash est un prompt pré-configuré qui guide Claude pour effectuer une série d'actions automatisées.Each Magic Slash command is a pre-configured prompt that guides Claude to perform a series of automated actions.

Serveurs MCP (Model Context Protocol)MCP Servers (Model Context Protocol)

Les serveurs MCP permettent à Claude d'interagir avec des services externes. Magic Slash configure deux serveurs :MCP servers allow Claude to interact with external services. Magic Slash configures two servers:

  • MCP Atlassian — Permet de lire les tickets Jira, récupérer les détails, mettre à jour les statuts, et interagir avec ConfluenceReads Jira tickets, fetches details, updates statuses, and interacts with Confluence
  • MCP GitHub — Permet de créer des Pull Requests et gérer les repositoriesCreates Pull Requests and manages repositories

FAQ / TroubleshootingFAQ / Troubleshooting

Comment autoriser le MCP Atlassian ?How to authorize the Atlassian MCP?

Lors de la première utilisation d'un outil Atlassian (Jira ou Confluence), Claude Code vous demandera d'autoriser l'accès. Un lien s'affichera dans le terminal : cliquez dessus pour ouvrir votre navigateur et connectez-vous avec votre compte Atlassian. Une fois l'autorisation accordée, le token OAuth sera automatiquement sauvegardé.

On the first use of an Atlassian tool (Jira or Confluence), Claude Code will ask you to authorize access. A link will appear in the terminal: click on it to open your browser and log in with your Atlassian account. Once authorization is granted, the OAuth token will be automatically saved.

Comment générer un token GitHub (Personal Access Token) ?How to generate a GitHub Personal Access Token?

Allez sur GitHub > Settings > Developer settings > Personal access tokens > Tokens (classic). Créez un nouveau token avec les permissions repo (accès complet aux repositories). Copiez le token, il ne sera affiché qu'une seule fois.

Go to GitHub > Settings > Developer settings > Personal access tokens > Tokens (classic). Create a new token with repo permissions (full repository access). Copy the token, it will only be displayed once.

Erreur "MCP server not found""MCP server not found" error

Cette erreur indique que Node.js n'est pas correctement installé ou n'est pas dans le PATH. Vérifiez avec node --version que Node.js v18+ est installé. Si ce n'est pas le cas, installez-le via brew install node ou depuis nodejs.org.

This error indicates that Node.js is not properly installed or not in the PATH. Check with node --version that Node.js v18+ is installed. If not, install it via brew install node or from nodejs.org.

Le worktree ne se crée pasWorktree doesn't get created

Vérifiez que vous avez les droits d'écriture sur le dossier parent des repositories. Assurez-vous aussi que le chemin dans ~/.config/magic-slash/config.json est correct et que le repository Git existe bien à cet emplacement.

Check that you have write permissions on the parent folder of the repositories. Also make sure the path in ~/.config/magic-slash/config.json is correct and the Git repository exists at that location.

Le ticket Jira n'est pas trouvéJira ticket not found

Vérifiez que l'URL Jira dans la configuration est correcte (format: https://votre-domaine.atlassian.net). Assurez-vous que vos credentials (email + API token) sont valides et que vous avez accès au projet contenant le ticket.

Check that the Jira URL in the configuration is correct (format: https://your-domain.atlassian.net). Make sure your credentials (email + API token) are valid and you have access to the project containing the ticket.

Le statut Jira ne se met pas à jourJira status doesn't update

Le workflow Jira peut restreindre certaines transitions de statut. Vérifiez dans Jira que la transition vers "To be reviewed" est autorisée depuis le statut actuel du ticket. Contactez votre administrateur Jira si nécessaire.

The Jira workflow may restrict certain status transitions. Check in Jira that the transition to "To be reviewed" is allowed from the ticket's current status. Contact your Jira administrator if needed.

Le commit échoue avec une erreur pre-commit hookCommit fails with a pre-commit hook error

Les hooks pre-commit peuvent bloquer un commit si le code ne respecte pas certaines règles (linting, formatage, tests). Lisez le message d'erreur pour identifier le problème, puis corrigez-le (ex: npm run lint:fix ou npm run format). Relancez ensuite /commit. Si vous devez absolument bypasser le hook, utilisez git commit --no-verify, mais c'est déconseillé.

Pre-commit hooks can block a commit if the code doesn't follow certain rules (linting, formatting, tests). Read the error message to identify the problem, then fix it (e.g., npm run lint:fix or npm run format). Then run /commit again. If you absolutely need to bypass the hook, use git commit --no-verify, but this is not recommended.

Simply type magic-slash in your terminal. An interactive menu displays your current configuration. Use ↑/↓ arrows to navigate and Enter to select an option.

La commande magic-slash n'est pas trouvéemagic-slash command not found

Vérifiez que ~/.local/bin est dans votre PATH. Ajoutez cette ligne à votre fichier de configuration shell (~/.bashrc, ~/.zshrc) : export PATH="$HOME/.local/bin:$PATH". Puis relancez votre terminal ou exécutez source ~/.bashrc.

Make sure ~/.local/bin is in your PATH. Add this line to your shell config file (~/.bashrc, ~/.zshrc): export PATH="$HOME/.local/bin:$PATH". Then restart your terminal or run source ~/.bashrc.

Conventional Commits

Magic Slash utilise le format Conventional Commits pour générer automatiquement des messages de commit cohérents et lisibles.

Magic Slash uses the Conventional Commits format to automatically generate consistent and readable commit messages.

Format

type(scope): description

Types disponibles

Available types

feat Nouvelle fonctionnalité — Ajout d'une nouvelle feature au produit New feature — Adding a new feature to the product
fix Correction de bug — Résolution d'un problème existant Bug fix — Fixing an existing issue
docs Documentation — Modifications de la documentation uniquement Documentation — Documentation changes only
style Formatage — Changements de style (sans modification de la logique) Formatting — Style changes (no logic modification)
refactor Refactoring — Restructuration du code sans changer le comportement Refactoring — Code restructuring without changing behavior
test Tests — Ajout ou modification de tests Tests — Adding or modifying tests
chore Maintenance — Tâches de maintenance, dépendances, config Maintenance — Maintenance tasks, dependencies, config

Exemples

Examples

feat(auth): add OAuth2 login with Google
fix(api): resolve null pointer in user endpoint
docs(readme): update installation instructions
refactor(utils): simplify date formatting logic
chore(deps): upgrade React to v18.2
